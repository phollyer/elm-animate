This measures how many frames your browser renders per second using requestAnimationFrame.

What This Means for elm-animate

The toCmd and toTask scroll functions assume a 60 FPS refresh rate when calculating animation frames.

  • ~60 FPS: Scroll durations will match the specified time.
  • ~120 FPS: Scrolls will complete in half the specified duration.
  • ~144 FPS: Scrolls will complete in ~42% of the specified duration.

For refresh-rate-independent animations, use the Sub engine instead - it uses delta-time interpolation for accurate timing.
