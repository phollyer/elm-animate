# 3D Animations

Elm Animate supports 3D transforms for Rotate, Scale, and Translate properties. 3D animations are GPU-accelerated and enable rich, immersive effects.

## Perspective

3D animations require perspective to create depth. Without perspective, they will appear flat.

Perspective is set by a `style` attribute on the parent element of the element being 3D animated. Any parent will do. Therefore, we can set perspective once at the
root level, on the direct parent of each 3D animated element, or however we choose within our page.

!!! note "Container Selection"
    - For typical UI animations (card flips, hover effects, etc.), you almost always want per-container perspective. A card in the corner of the page should flip the same way as one in the center.
    - For immersive 3D scenes where you want depth across the whole page, root-level perspective makes sense.

There are two ways to set the perspective:

- Manually add the `style` attribute: `Html.Attributes.style "perspective" "1000px"`
- Use the helper functions in each Engine.

Perspective can be set globally on the Engine, or locally on a property - local always wins.

```elm
import Anim.Property.Translate as Translate

translate3D builder =
    builder
        |> Translate.for "box"
        |> Translate.perspective "container" 1000  -- 1000px local perspective depth
        |> Translate.fromZ 0
        |> Translate.toZ 200
        |> Translate.build

animateFunction animState =
    animaState 
        |> CSS.builder
        |> CSS.perspective "container1" 800 -- 800px global perspective depth
        |> translate3D
        |> ... other element or property animations
        |> CSS.animate
```

### Perspective Values

| Value | Effect |
| ------- | -------- |
| 500-800px | Dramatic, close-up 3D effect |
| 1000-1500px | Natural, balanced perspective |
| 2000px+ | Subtle, distant 3D effect |

## View Setup

In order for the browser to render elements in 3D, a parent element (any parent) needs
to have the perspective style attached.

```elm
view model =
    div [ id "container" ]  -- Perspective applied here
        [ div 
            ([ id "card" ] ++ CSS.transitionAttributes "card" model.animation)
            [ text "I flip in 3D!" ]
        ]
```

!!! note "Perspective container"
    Perspective can be set manually with a `style` attribute: `Html.Attributes.style "perspective" "1000px"`, or you can use a helper function from your selected Engine.


## Translate in 3D

Move elements along the Z-axis (towards/away from the viewer).

```elm
import Anim.Property.Translate as Translate

-- Move element towards viewer
popOut builder =
    builder
        |> Translate.for "card"
        |> Translate.perspective "container" 1000
        |> Translate.fromZ 0
        |> Translate.toZ 100
        |> Translate.build

-- Full 3D movement
flyIn builder =
    builder
        |> Translate.for "card"
        |> Translate.perspective "container" 1000
        |> Translate.fromXYZ -200 -100 -500
        |> Translate.toXYZ 0 0 0
        |> Translate.build
```

**Remember**: Any time you animate the `Z` axis, you need to add `perspective`.

## Rotate in 3D

Rotate elements around any axis for card flips, barrel rolls, and more.

```elm
import Anim.Property.Rotate as Rotate

-- Flip a card horizontally (like turning a page)
flipCard builder =
    builder
        |> Rotate.for "card"
        |> Rotate.perspective "container" 1000
        |> Rotate.fromY 0
        |> Rotate.toY 180
        |> Rotate.build

-- Flip vertically (like a coin toss)
coinFlip builder =
    builder
        |> Rotate.for "coin"
        |> Rotate.perspective "container" 800
        |> Rotate.fromX 0
        |> Rotate.toX 360
        |> Rotate.build

-- Barrel roll
barrelRoll builder =
    builder
        |> Rotate.for "plane"
        |> Rotate.perspective "container" 1000
        |> Rotate.fromXYZ 0 0 0
        |> Rotate.toXYZ 360 180 360
        |> Rotate.build
```

**Remember**: Any time you rotate on the `X` or `Y` axes, you need to add `perspective`

## Scale in 3D

Scale elements along the Z-axis for depth effects.

```elm
import Anim.Property.Scale as Scale

-- Scale in depth
depthScale builder =
    builder
        |> Scale.for "box"
        |> Scale.perspective "container" 1000
        |> Scale.fromZ 1
        |> Scale.toZ 2
        |> Scale.build

-- Full 3D scale
grow3D builder =
    builder
        |> Scale.for "box"
        |> Scale.perspective "container" 1000
        |> Scale.fromXYZ 0.5 0.5 0.5
        |> Scale.toXYZ 1 1 1
        |> Scale.build
```

**Remember**: Any time you scale the `Z` axis, you need to add perspective

## Combining 3D Transforms

Combine multiple 3D properties for complex effects:

```elm
cardFlipWithZoom : AnimBuilder -> AnimBuilder
cardFlipWithZoom builder =
    builder
        |> Rotate.for "card"
        |> Rotate.perspective "container" 1000
        |> Rotate.fromY 0
        |> Rotate.toY 180
        |> Rotate.build
        |> Scale.for "card"
        |> Scale.from 1
        |> Scale.to 1.2
        |> Scale.build
        |> Translate.for "card"
        |> Translate.fromZ 0
        |> Translate.toZ 50
        |> Translate.build
```

!!! warning "Backface visibility"
    When flipping elements 180°, you may want to hide the back face. Add `style "backface-visibility" "hidden"` to your element.

## Performance Tips

!!! tip "GPU Acceleration"
    All 3D transforms are GPU-accelerated. They're as performant as 2D transforms.

!!! tip "Avoid layout thrashing"
    Keep perspective containers stable. Changing container dimensions during animation can cause performance issues.

!!! tip "Use transform-style"
    For nested 3D elements, add `style "transform-style" "preserve-3d"` to maintain 3D context.
